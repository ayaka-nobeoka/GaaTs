# 今日の覚えたこと
## Githubでリポジトリを作るには
※GitとGitHUbは同じものではなく、gitHubはGitを管理するためのツールだった！

①Githubでリポジトリを作る
②同じ名前でローカル(自分のPC)でファイルを作る
③ターミナルでそのファイルにいく(cdコマンドのやつ)
④そこからgitのセットアップ(コマンドをうっていく)
⑤ローカルとリモートで繋がったらそこから作業ブランチを作って行ったりする(main→develop→feater/00/名前 みたいに)

## ポイント
githubのリポジトリの名前とファイルの名前を一緒にしないとリモートで作成したものろローカルで作成したものが何か分からなくなってしまわないよう注意！！

## なんでコンフリクトが起きるの？？
一人で作業していたら差分とか変更した内容を覚えているからいらないけど複数人で作業していたら同じファイルを編集を同時にやってしまった場合編集が被ってしまう(この同じ箇所で発生した差分のことをコンフリクトという)のを管理するためにブランチを切ってコンフリクトが起きたらどっちにしましょうかって教えてくれるイベント？がコンフリクト！（英訳が衝突だった）

## 解決できるコマンドが決まっててある
まずはdevelopブランチに移動します。その後pullします。
そしてfeature/greenに移動してdevelopブランチにgit mergeします。
training-git % git checkout feature/green
Switched to branch 'feature/green'

## リポジトリをローカルと繋げられたら
add commit pushする(いつも研修のときに課題の最後にやるやつ)

そこまでしたらgithubに行ってmergeをしてプルリクを出す
